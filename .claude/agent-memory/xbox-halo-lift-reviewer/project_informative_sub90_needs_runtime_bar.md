A clean, fully-classified static body audit does NOT clear AUTO_ACCEPT when VC71 is **informative and below 90%**. The <90% band literally requires "golden/runtime behavior verification PLUS classified mismatches" — classified mismatches alone are the necessary half, not the sufficient half. Verdict = NEEDS_RUNTIME (not REJECT when no concrete bug + ABI complete + no FPU risk).

**Why:** The disasm-audit bars in memory ([[project_real_math_putLong_eax_ebx_disasm_audit_bar]], gz_destroy, gz_flush, deflateInit2_) all cleared on static-alone *because VC71 was BLIND* — stale/short stub, unregistered ref, or uninformative % (2.5%, 54-insn stub). That is the no-signal fallback. When VC71 is **informative** (e.g. 78.7% against a valid, registered `delinked/functions/<addr>.obj`), the band policy governs and static alone is insufficient. Even the FPU-leaf cap that WAS legitimately capped (cc90/vector_intersects_pill2d, [[project_real_math_fpu_equiv_activation_bar]]) still required EQUIVALENCE, never static-only.

**How to apply:**
- Check the band FIRST and quote the RIGHT band. 78.7% is **<90%**, not 90-95%. The 90-95% requirements (classify + call-audit + offset-audit + no-Uncertain) are NOT the bar for <90% — they're a subset of it.
- Do NOT report "157/158, one delta" as if that's the match. Counts (157/158) differ by 1; the **LCS match is 78.7%**. One redundant ref-only reload explains the COUNT delta, not a ~21pp gap. The gap = regalloc reordering of semantically-equivalent insns + the @<reg>-DEFINED prologue cap, manually reconciled. Saying "1 delta" overstates structural match and the next reviewer can't reconcile it against 78.7%.
- Integer-only + cdecl-stubbable callees = textbook "structurally capped → equivalence to prove behavior." Recommend `unicorn_diff.py <addr> --allow-stubs --mem-trace`; caveat that state-pointer-walking fns (head[]/prev[]/window) need `--state-snapshot` ([[reference_statesnapshot_recovers_vc71capped]]) because zero-fill under-covers the slide+rebase path.
- A high-confidence equivalence pass would make AUTO_ACCEPT defensible on RE-review; until it exists, NEEDS_RUNTIME.

**Worked case (FUN_00111770, zlib fill_window, real_math.obj, @<esi>=state, 78.7%):** body audit was fully clean — 7-point verify all PASS (more special-paths, slide cond 2*w_size-0x106 unsigned, both rebase loops head@0x3c/prev@0x38 not swapped, assert `jae` unsigned, adler `FUN_00110a10(adler,next_in,n)` guarded by wrap==0, csmemcpy(dest,next_in,n), ins_h dual-store @0x40, w_size read ONCE at prologue 0x09 never reloaded in loop). No FPU, no CONCAT, no NULL-reg-arg, callees all cdecl. Still NEEDS_RUNTIME because informative sub-90 with no runtime evidence.

**Worked case #3 (FUN_0018ffe0, wind_update, wind.obj (NEW obj, binary evidence c:\halo\SOURCE\scenario\wind.c — task header guessed scenario.obj wrongly), cdecl regs=NONE, 86.3% = 182/189, per-fn ref delinked/wind_FUN_0018ffe0.obj REGISTERED with correct FUN_0018ffe0 symbol → informative):** the trap here is that EQUIVALENCE ALREADY RAN and PASSED (100/100 seeds, 0 arg mismatch, stub-arg differential 200 calls/0 mismatch) — yet it STILL does not clear the runtime bar, because coverage was **8.4% / weak / "only early-exit path."** The loop `for(i=0;i<*block;i++)` over the wind palette: with zero-fill, *block (scenario+0x1b4 count) = 0, so the loop body never runs. The passing seeds exercised only prologue + empty-loop tail (count store to 0x5060c4). The ENTIRE record-update body that carries all the risk — the ±0.01 random-walk clamps (t@0x4 [0,1], yaw@0xc [-1,1], pitch@0x8 [-1,1]), velocity lerp (wind[1]-wind[0])*t+wind[0], vector_to_angles/angles_to_vector with yaw/pitch perturb by wind[3]/wind[2]*perturb*0.5, direction scale by elem+0x9c*velocity — was NEVER executed at runtime. Lesson: **a passing equivalence run at weak coverage is NOT runtime verification of the body**; check the coverage %/confidence tier, not just pass/fail. Cheap definitive next step: `unicorn_diff.py 0x18ffe0 --allow-stubs --mem-trace --state-snapshot <snap with a non-empty wind palette>` (or golden test) to drive the loop ≥1 iteration. My full static body audit WAS clean (every offset/arg/clamp-polarity/FPU-order 1:1 vs reference disasm; ABI regs=none; no hazard/FPU-WARN). One benign wrinkle: the assert-abort at the `wind_globals.initialized==0` path — candidate calls `system_exit`@0x8e2f0(-1), original calls FUN_001029a0@0x1029a0(-1) (kb labels 0x1029a0 `halt_and_catch_fire(void)` yet the original PUSHES -1 there → kb ABI-label curiosity). Different fatal target but non-gameplay assert-only path, both fatal → benign, non-blocking, contributes to the 7-insn gap along with cold-block layout. Verdict NEEDS_RUNTIME.

**Worked case #2 (FUN_0018ef30, scenario_debug_dump_status, scenario.obj, cdecl regs=NONE, 85.8% = 104/108, per-fn ref delinked/functions/0018ef30.obj UNREGISTERED in objdiff.json — compare_obj directly):** the STRONGEST-possible static case that still does NOT clear the gate. Full 1:1 disasm audit of the reference was byte-for-byte clean: deferred-vararg scheduling assembles correctly (bsp_index pushed @0x45 + first tag_get_name result survive cleanup as the `%s bsp %s (#%d)` varargs), buffer-alias §5 handled (`player 0x%08x` reads iter.datum_handle @ebp-0x8=iter+0x8, NOT the returned element), every offset verified (bsp_index int16@0x326a0c, pos x/y/z float@obj+0x50/0x54/0x58, leaf dword@obj+0x48, cluster int16@obj+0x4c, handle@element+0x34), type const object_get_and_verify_type(h,3), tag_block_get_element(global_scenario+0x5a4,idx,0x20) 3 args. ABI audit regs=none clean. No FPU arithmetic (only flds/fstpl double-promotion into fprintf — no operand-order risk). Hazard scan's only flag (line 410 matrix_transform_point dup-arg) is OUTSIDE the lifted range (~1298-1371) → pre-existing, non-blocking. **The entire 4-insn gap is benign cold-block placement**: clang inlines the `<no scenario loaded>` early-return near the top; MSVC hoists it to end (LAB_0018f05b) falling through into the shared epilogue (LAB_0018f06c). Lesson: regs=NONE cdecl means there is NO prologue-cap excuse — the score is genuinely recoverable (permuter can reorder blocks) or provably benign, yet <90% STILL mandates runtime. A clean static read ≠ golden/runtime. Verdict NEEDS_RUNTIME (no concrete bug → not REJECT).
