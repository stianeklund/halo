# Memory policy

Store only durable, non-obvious information useful for future Halo/Xbox RE.

Save:
- confirmed recurring RE patterns in this XBE
- durable subsystem boundaries
- durable global meanings or helper conventions
- durable user collaboration preferences
- durable references to tools or external systems

Do not save:
- repo structure
- recent file changes
- temporary task state
- per-target scratch notes
- current conversation bookkeeping
- ephemeral fix recipes that belong in code or commits

Memory is contextual, not authoritative.
If memory conflicts with current evidence, trust current evidence and update
or remove stale memory.
